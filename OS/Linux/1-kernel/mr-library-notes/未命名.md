### 🧰 核心 API 兵器谱（按使用频率与重要性排序）

这些 API 涵盖了 LDD3 中的核心章节。你可以将以下编号视为点播台。**后续会话中，你只需发送对应的编号（如 `#API-04`），我将立即为你展开该模块的第一性原理、静态数据结构 UML、动态执行序列图以及费曼式详解。**

#### 🥇 优先级 1：基础生命线（没有它们驱动寸步难行）

**【#API-01】用户与内核边界的数据搬运工 (Data Transfer)**

- **关联概念：** MMU 隔离、安全边界。
- **核心 API：** `copy_to_user()`, `copy_from_user()`
- **入选理由：** 任何驱动的 `read` 和 `write` 函数必备。内核态绝对不能直接解引用用户态指针。

**【#API-02】内核内存管家 (Memory Allocation)**

- **关联概念：** LDD3 Chapter 8: Allocating Memory、Slab 分配器、物理连续与虚拟连续。
- **核心 API：** `kmalloc()`, `kzalloc()`, `kfree()`, `vmalloc()`, `vfree()`
- **入选理由：** 驱动私有上下文（`private_data`）和缓冲区的生命周期基石。

#### 🥈 优先级 2：硬件交互与防护（驱动工程师的核心饭碗）

**【#API-03】并发防御与锁机制 (Concurrency & Race Conditions)**

- **关联概念：** LDD3 Chapter 5: Concurrency、死锁、抢占。
- **核心 API：**
    - 互斥量：`mutex_init()`, `mutex_lock()`, `mutex_unlock()`
    - 自旋锁：`spin_lock_init()`, `spin_lock_irqsave()`, `spin_unlock_irqrestore()`
- **入选理由：** 只要你的设备能被多个用户同时 `open`，或者存在中断，就必须加锁。

**【#API-04】阻塞 I/O 与等待队列 (Advanced Char & Wait Queues)**

- **关联概念：** LDD3 Chapter 6: Advanced Char Driver Operations、进程休眠与唤醒、上下文切换。
- **核心 API：** `init_waitqueue_head()`, `wait_event_interruptible()`, `wake_up_interruptible()`
- **入选理由：** 这是你此次点名的切入点！当用户读数据但硬件还没准备好时，如何让用户进程优雅地“睡去”，并在数据到来时“秒醒”。

**【#API-05】硬件寄存器映射操作 (Memory Mapping)**

- **关联概念：** LDD3 Chapter 15: Memory Mapping、I/O 内存、非缓存映射。
- **核心 API：** `ioremap()`, `iounmap()`, `readl()`, `writel()`
- **入选理由：** Linux 内核开启了 MMU，驱动无法直接访问物理地址 0x40001000。必须通过重映射，才能实现点灯、控制寄存器。

#### 🥉 优先级 3：异步事件与时间管理（打造高性能驱动）

**【#API-06】硬件中断处理 (Interrupt Handling)**

- **关联概念：** LDD3 Chapter 10: Interrupt Handling、硬中断与软中断。
- **核心 API：** `request_irq()`, `free_irq()`, 极简的中断服务例程 (ISR) 编写规范。
- **入选理由：** 硬件主动通知 CPU 的唯一合法途径。理解 UIO 和 ipc-shm 高效跨核通信的前置知识。

**【#API-07】时间、延迟与下半部机制 (Time, Delays & Deferred Work)**

- **关联概念：** LDD3 Chapter 7: Time, Delays, and Deferred Work、滴答定时器、任务延期。
- **核心 API：**
    - 时间与延迟：`jiffies`, `msleep()`, `ndelay()`
    - 工作队列（中断下半部）：`INIT_WORK()`, `schedule_work()`
- **入选理由：** 中断里不能睡觉，耗时操作怎么处理？如何让硬件等 10 毫秒后再启动？全靠这套机制。
