地址边界               容量                  Cortex-A 物理内存映射 (异构兼容规范)
================================================================================================

                                         🚀 【64位大航海时代：Cortex-A 独占扩展区】
0xFF_FFFF_FFFF +---------------------------+
               |                           |  
               |                           |  - Cortex-A 专属主内存海洋。
               |   Extended DRAM (TB级)    |  - Linux 内核运行、MMU 开启后海量虚拟地址映射的
               |                           |    真实物理后盾。
               |                           |  
0x08_0000_0000 +---------------------------+  ------------------------------------------------
               |                           |  
               |  Extended Device (28GB)   |  - Cortex-A 专属高速外设区。
               |                           |  - 用于映射需要海量寻址空间的高速 PCIe 设备等。
               |                           |  
0x01_0000_0000 +---------------------------+================================================
                                         
                                         ⬇️ 【32位向下兼容边界：前 4GB 妥协与共享区】
0x00_FFFF_FFFF +---------------------------+
               |                           |  📦 【32位物理内存门槛】 (Lower DRAM)
               |    Lower DRAM (2.0GB)     |  - A系列独有妥协：为了照顾 32 位的 OS 与老旧 DMA。
               |                           |  - 必须在 4GB 范围内留出一段物理 DRAM 作为起步区。
0x00_8000_0000 +---------------------------+  ------------------------------------------------
               |                           |  🎛️ 【片上外设与低速 IO】 (Peripherals)
               |   Peripherals (1.0GB)     |  - [与 M 系列完全一致] A/M 核共同操作的硬件开关。
               |                           |  - A53 的 GIC 中断控制器通常也会从 PPB 搬移到此处。
0x00_4000_0000 +---------------------------+  ------------------------------------------------
               |                           |  ⚡ 【片上 SRAM / 共享内存】 (On-chip RAM)
               |      SRAM     (0.5GB)     |  - [与 M 系列完全一致] 异构核间通讯的零拷贝工作台。
               |                           |  - A53 与 M7 在这里通过 IPC (如 RPMsg) 交换数据。
0x00_2000_0000 +---------------------------+  ------------------------------------------------
               |                           |  📚 【启动与安全区】 (Boot ROM / Trusted Flash)
               |   Boot / Code (0.5GB)     |  - [与 M 系列完全一致] 芯片的物理原点。
               |                           |  - 存放出厂死锁的 Boot ROM、以及提供验签的安全固件。
0x00_0000_0000 +---------------------------+
