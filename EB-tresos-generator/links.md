在 `.link` 文件中，你不需要为每个版本的插件单独编写配置。`.link` 文件的作用仅仅是指定一个**外部根目录**，EB 框架会自动去该目录下的 `eclipse/plugins` 文件夹中加载所有合法的插件。

为了并行加载两个版本的 IPCF 插件，请按以下格式和步骤操作：

1. **创建统一的外部目录结构**：在电脑任意位置创建一个根目录，并在其下严格按照 Eclipse 规范创建 `eclipse/plugins` 子文件夹（例如 `C:/my_external_plugins/eclipse/plugins`）。
2. **放入两个版本的插件**：将已经修改好不同 `Bundle-SymbolicName` 和文件夹名称的两个 IPCF 插件文件夹，**同时**放入上述的 `plugins` 文件夹中。
3. **编写 .link 文件**：在 EB tresos 安装目录的 `links` 文件夹下创建或编辑一个 `.link` 文件（例如 `ipcf.link`）。
4. **写入指向根目录的路径**：在文件中写入以下内容，**必须严格使用正斜杠 `/`**：
    
    ```
    path=C:/my_external_plugins
    ```
    
    _（注意：路径只写到根目录，千万不要包含后半段的 `eclipse/plugins`）_

配置完成后重启 EB tresos，系统就会从这个外部路径同时加载 0.8 和 1.0 两个版本的插件了。
