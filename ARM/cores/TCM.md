# 架构笔记：多核 MCU 的 TCM 私有化与全局映射机制

## 一、 核心心智模型：同构多核的 TCM 悖论

- **现象：** 在同构多核 SOC（如双核 Cortex-M7）中，Core A 和 Core B 访问自己 TCM 的物理地址是一模一样的（例如都是 `0x2000_0000`）。
    
- **本质：物理绝对私有，逻辑完美重叠。**
    
- **类比记忆：** 就像计算机网络中的 `127.0.0.1`（localhost）。两台电脑 ping 这个地址，访问的都是各自的本地网卡，永远不会在外部路由器（总线矩阵）上发生冲突。
    
- **架构收益：** 极致的代码复用。同一个编译出来的 `.elf` 固件和链接脚本（Linker Script），可以直接丢给两个核运行，无需修改地址偏移。

## 三、 RAM 映射机制：双重视角（Aliasing）

既然 TCM 被内部拦截了，DMA 或 Core A 如何去读 Core B 的数据（实现跨核 IPC 或外设数据搬运）？这就依赖于芯片层面的**双重视角映射（Global Backdoor）**。

同一块物理 TCM 硅片，在整个 SOC 的内存映射表（Memory Map）中拥有**两个不同的物理地址**：

1. **第一视角（本地私有视图 Local View）：**
    
    - **地址特征：** 统一且重叠（例如 Core A 和 Core B 看自己都是 `0x2000_0000`）。
        
    - **硬件通路：** 走 CPU 内部专线，经 Local Decoder 拦截。
        
    - **特性：** 极速，0 等待周期，不发生总线仲裁。
        
    - **用途：** 核心自己跑算法、存放自身栈空间（Stack）和高频中断向量表。
        
2. **第二视角（全局后门视图 Global View）：**
    
    - **地址特征：** 绝对唯一（例如在全局内存表中，Core A 的 TCM 映射在 `0x4000_0000`，Core B 的映射在 `0x4010_0000`）。
        
    - **硬件通路：** 走外围的全局 AXI/AHB 总线矩阵，通过芯片原厂开的“后门从属端口（Backdoor Slave Port）”进入对应的 TCM。
        
    - **特性：** 较慢，需要参与全局总线仲裁（会与访问 Flash / SRAM 的请求排队）。
        
    - **用途：** 将此地址配置给 DMA 指针进行数据搬运，或者用于不同核心之间的共享内存通信。
        

> **最佳实践警告：** CPU 自身运算时永远用第一视角地址；但只要把数据地址交给 DMA 或者其他核心，**必须**进行指针转换，传递第二视角（全局）地址。
