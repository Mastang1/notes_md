**devtmpfs** 的全英文称是 **Device Temporary File System**。

它的中文通常被称为 **设备临时文件系统**。

### **补充说明**

为了让你更好地理解它的作用，这里简单介绍一下它的背景：

- **功能定位：** 在 Linux 系统中，`devtmpfs` 是一个直接由内核在内存中维护的文件系统，通常被挂载在 `/dev` 目录下。
    
- **为什么需要它：** 在 `devtmpfs` 引入内核之前，`/dev` 目录下的设备节点（如 `sda`, `tty` 等）主要依靠用户空间的守护进程（如 `udev` 或早期的 `mdev`、`devfs`）来动态创建。这会导致在系统启动的极早期（用户空间程序尚未运行前），内核没有现成的设备节点可用。
    
- **它的优势：** 引入 `devtmpfs` 后，内核驱动程序在注册设备时，内核会自动在这个临时文件系统中创建对应的设备节点。这不仅大幅加快了系统的启动速度，还简化了早期的系统初始化流程。当用户空间的 `udev` 启动后，它会在 `devtmpfs` 的基础上接管权限管理和创建更复杂的软链接。
