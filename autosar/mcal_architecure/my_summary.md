
## 1. 这是个驱动一般用的分层架构

- 上层 API 层（对应用/中间件）
- 服务/功能层（缓冲池管理、队列管理、协议处理等）
- 硬件抽象层（HAL：寄存器访问、中断控制）
- OS 抽象层（OSAL：任务/中断/事件接口）

## 2. 架构视图（架构概览)
 - 这个视图和章节的重点不只在于分层，核心是为了向开发人员展示软件的上下文边界，内部的简单分层

## 3. 组件视图
 - *组件*：判断依据 接口界定清晰、功能独立可复用，下层依赖明确，具体参考arch-*  的几个文档
 - 组件视图就是把软件系统展开为组件的视图
## 4. 模块
 -  *模块*： 模块偏向于代码实现，一般会把某个文件中实现的静态代码作为一个模块，属于详细设计的概念
