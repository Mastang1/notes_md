# 实时目标检测算法（YOLO 及同类模型）

## 1. 核心概念与功能
这类算法的核心任务是实现**单阶段（One-Stage）实时视觉感知**。基于单目相机输入，模型在一次前向传播中同时输出以下信息：

*   **类别 ID (Class ID)**：物体是什么（如 `0: person`, `1: car`）。
*   **置信度 (Confidence Score)**：模型对该确定性的概率评估（$0 \sim 1.0$）。
*   **位置信息 (Spatial Bounding Box)**：
    *   *2D 检测（默认）*：图像像素坐标系下的边界框 `[x, y, w, h]`。
    *   *3D 检测（扩展）*：真实三维空间中的相机坐标 `[X_m, Y_m, Z_m]` 及朝向。

---

## 2. 主流模型对比

| 模型家族 | 技术特点 | 核心优势 | 劣势/局限 |
| :--- | :--- | :--- | :--- |
| **YOLO 系列** *(v5 ~ v12)* | CNN / 区域注意力机制，生态极度完善 | 边缘端部署生态极好，速度与精度的行业标杆 | 历史版本过多，特殊硬件的算子兼容需调优 |
| **RT-DETR** | 基于 Transformer 架构，端到端预测 | 精度上限高，无需 NMS 后处理，避免重叠漏检 | 在极低算力芯片（如低端单片机）上优化难度大 |
| **FCOS / CenterNet** | Anchor-Free（无锚框）结构，基于中心点 | 结构优雅，对尺度变化鲁棒，适合高密度重叠场景 | 小物体检测相对弱于带 Anchor 的精调模型 |

## 3. 典型应用场景与用途

### 🏃 场景 A：工业自动化与边缘分拣（Speed-Centric）
*   **具体用途**：流水线缺陷检测、机械臂抓取引导、传送带物料计数。
*   **核心诉求**：极低延迟（$< 10\text{ ms}$）、高帧率。
*   **模型选型**：YOLOv8-Nano / YOLOv12-N 等轻量化版本，通常转换为 TensorRT 或 NCNN 运行在边缘算力板卡上。

### 🚗 场景 B：智能交通与自动驾驶（Accuracy & 3D-Centric）
*   **具体用途**：前向碰撞预警（FCW）、盲区监测（BSD）、行人轨迹预测。
*   **核心诉求**：必须将 2D 像素框转化为 3D 真实世界距离，且对漏报（FN）容忍度极低。
*   **模型选型**：RT-DETR 或 融合了深度估计的 YOLO-3D 变体。

### 📹 场景 C：智能安防与行为分析（Tracking-Centric）
*   **具体用途**：区域入侵检测、客流计数、未戴安全帽/口罩合规检查。
*   **核心诉求**：多通道视频流并发处理能力，需配合下一级时序/追踪算法。
*   **模型选型**：YOLO 标准版 + ByteTrack / DeepSORT 追踪器。
