# 📝 Linux 虚拟文件系统 (Pseudo Filesystems) 核心笔记

> 📌 **适用内核**：Linux Kernel 5.10+ (持续活跃至今)
> 
> 💡 **核心概念**：这些“文件夹”里并没有真正的文件，它们不占硬盘空间，而是存活在**内存**中。Linux 把计算机的“五脏六腑”（硬件状态、运行日志、系统设置）巧妙地伪装成了文件，让我们用最简单的读写操作就能控制整个系统。

### 📊 核心文件系统清单（按使用优先级 / 重要度排序）

| **优先级**                    | **文件系统**         | **默认挂载点**                      | **费曼通俗解释 (它就像什么？) & 核心作用**                                                                                     | **常见文件 / 目录示例**                                   |
| -------------------------- | ---------------- | ------------------------------ | -------------------------------------------------------------------------------------------------------------- | ------------------------------------------------- |
| **P0**<br><br>  <br><br>必备 | **`procfs`**     | `/proc`                        | **💻 系统的“任务管理器”**。<br><br>  <br><br>这里展示了当前系统正在运行的所有程序（进程）和整体健康状况。你想知道现在剩多少内存、CPU 型号是什么，看这里就对了。                | `/proc/meminfo` (看内存)`/proc/1/` (1号进程的信息)         |
| **P0**<br><br>  <br><br>必备 | **`sysfs`**      | `/sys`                         | **🎛️ 硬件的“控制面板”**。<br><br>  <br><br>内核把所有的硬件设备整理成了一本清晰的目录。你可以看看插了哪些设备，或者往里面写个数字来控制硬件（比如调节屏幕亮度或 CPU 功耗）。        | `/sys/class/net/` (网卡列表)`/sys/power/state` (控制休眠) |
| **P1**<br><br>  <br><br>高频 | **`tmpfs`**      | `/tmp`<br><br>  <br><br>`/run` | **📝 纯内存的“临时白板”**。<br><br>  <br><br>完全建立在内存上的极速空间。程序在这里写字非常快，但只要一停电（重启），白板上的字就会被全部擦除。适合放临时数据。                  | `/dev/shm` (程序间共享白板)`/run/user/1000` (当前用户状态)     |
| **P1**<br><br>  <br><br>高频 | **`devtmpfs`**   | `/dev`                         | **🔌 设备的“插座面板”**。<br><br>  <br><br>当你插上一个 U 盘或鼠标，系统会立刻在这里生成一个“插座”（设备节点）。软件必须接上这个插座，才能和你的 U 盘通信。                | `/dev/sda` (第一块硬盘)`/dev/null` (吃掉数据的黑洞)           |
| **P2**<br><br>  <br><br>进阶 | **`cgroup2`**    | `/sys/fs/cgroup`               | **🍰 资源的“分蛋糕机器”**。<br><br>  <br><br>用来限制某个程序最多能吃多少内存和 CPU。它是 Docker 和 K8s 容器技术的灵魂，确保没有程序能独占整台电脑。               | `/sys/fs/cgroup/cgroup.procs`                     |
| **P2**<br><br>  <br><br>进阶 | **`bpffs`**      | `/sys/fs/bpf`                  | **🎒 eBPF 超级程序的“随身背包”**。<br><br>  <br><br>现在 Linux 允许你在内核里跑微型程序（eBPF）。这个文件夹就是给这些微型程序存放数据的地方，防止它们退出时数据丢失。       | 现代防火墙/云网络工具数据驻留点                                  |
| **P3**<br><br>  <br><br>诊断 | **`tracefs`**    | `/sys/kernel/tracing`          | **📷 内核的“高速摄像机”**。<br><br>  <br><br>系统卡顿了？打开这个摄像机，它能以微秒级的速度，记录下系统大脑（内核）刚刚到底执行了什么动作，用来抓性能瓶颈。                    | `/sys/kernel/tracing/current_tracer` (开启抓拍)       |
| **P3**<br><br>  <br><br>诊断 | **`pstore`**     | `/sys/fs/pstore`               | **✈️ 系统崩溃的“黑匣子”**。<br><br>  <br><br>如果电脑突然死机（Panic），它会拼尽最后一口气，把死机原因写进主板上一块断电不丢失的芯片里。下次开机就能在这里查明死因。             | `/sys/fs/pstore/dmesg-ramoops` (死机遗言)             |
| **P4**<br><br>  <br><br>底层 | **`efivarfs`**   | `/sys/firmware/efi/...`        | **⚙️ 主板 BIOS 的“遥控器”**。<br><br>  <br><br>让你不需要重启狂按 F2 进 BIOS，直接在 Linux 里就能修改主板上的设置（比如修改下次开机用哪个硬盘启动）。            | 配置 UEFI 引导顺序的底层依赖                                 |
| **P4**<br><br>  <br><br>底层 | **`securityfs`** | `/sys/kernel/security`         | **🛡️ 保安的“规章制度本”**。<br><br>  <br><br>Linux 内部的安全警卫（如 SELinux 等）用来存放访客名单和放行规则的地方。                               | TPM 芯片接口                                          |
| **P4**<br><br>  <br><br>底层 | **`configfs`**   | `/sys/kernel/config`           | **🏭 虚拟硬件的“定制工厂”**。<br><br>  <br><br>跟 `sysfs` 相反。`sysfs` 是系统告诉你有什么硬件；而在这里，是你命令系统去凭空“捏造”一个虚拟硬件出来（比如虚拟出一个 U 盘）。 | 配置虚拟 USB 设备                                       |
| **P5**<br><br>  <br><br>开发 | **`debugfs`**    | `/sys/kernel/debug`            | **🔧 维修工的“凌乱操作台”**。<br><br>  <br><br>没有任何规范，专门给开发 Linux 源码的程序员准备的后门。普通用户千万别动，生产环境中通常也是锁死不让进的。                  | 底层驱动的深度调试文件                                       |
