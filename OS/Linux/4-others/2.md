# Linux 开机自动化：通俗原理解析与核心路径

要理解 Linux 的开机自启动，不要去死记硬背内核态、用户态等术语。你可以把 Linux 系统想象成一家**即将开业的餐厅**。从老板拿到钥匙开门，到最终顾客进店，必须经过四个严格的准备阶段。

## 各阶段详细动作与执行路径解析

### 阶段一：接通水电（基础文件系统挂载）
餐厅刚拿到钥匙，第一件事是把水管接通、电表拉好。如果你没有桌子，厨师来了也没法切菜。
* **具体文件路径：** `/etc/fstab`
* **核心操作：** 内核启动后，立刻读取这个文件。它负责把外部存储（SD卡、eMMC）和内存映射区（`tmpfs`）拼接到系统的目录树上。
* **为什么重要：** 这是整个系统最早执行的自动化动作。如果你的业务代码需要跑在 `tmpfs`（内存盘）里来提升速度，**必须**把挂载指令写在这里。写在后面可能会导致应用启动时找不到路径。

### 阶段二：设备通电与员工上岗（硬件与服务初始化）
水电通了，接下来要让冰柜运转（硬件识别），并让核心员工（后台服务）按顺序上岗。保安没来，门不能开；网络没通，收银机不能动。
* **动作 A：认硬件 (Udev)**
    * **文件路径：** `/etc/udev/rules.d/` (通常新建一个 `99-custom.rules`)
    * **核心操作：** 系统底层识别到物理线路有反应（比如你插了一个串口线或者 USB），立刻触发写在这里的规则。例如：`ACTION=="add", KERNEL=="ttyUSB0", RUN+="/bin/myscript.sh"`。
* **动作 B：大管家派活 (Init / Systemd)**
    * **文件路径：** `/etc/init.d/` (老式系统) 或 `/etc/systemd/system/` (现代系统)。
    * **核心操作：** 启动那些需要一直在后台活着、不能死掉的程序（Daemon 守护进程）。比如网络管理器、你的核心业务数据采集程序。大管家会监控它们，死了就重新拉起来。

### 阶段三：老板开业前的随手便利贴（自定义晚期脚本）
所有大中型设备都跑起来了。在正式开门迎客前，老板在门上贴了个便利贴，交代两句简单的杂活。
* **具体文件路径：** `/etc/rc.local` （或者通过 crontab 配置的 `@reboot` 任务）
* **核心操作：** 这是一个 Bash 脚本。通常用来放一些**一次性的、没有复杂依赖关系的**简单指令。
* **实战举例：** * 给工控板的蜂鸣器响一声：`echo 1 > /sys/class/gpio/gpio10/value`
    * 顺手拉起一个简单的数据同步脚本：`/opt/my_sync.sh &`
* **费曼提示：** 把它当成“万金油”。不知道写在哪里的简单开机命令，丢这里通常没问题，因为它执行得很晚，此时网络和硬盘肯定都已经准备好了。

### 阶段四：顾客进店与私人定制（用户登录交互）
正式开业。顾客（User）走进大门，服务员会递上标准的菜单，并根据老顾客的喜好做些微调。**（注意：如果你做的工控机是无人值守不插屏幕的，这一步就不会发生）**。
* **全局配置路径：** `/etc/profile` （所有人进店都会先拿到的通用菜单，用来设置全局 `PATH` 环境变量）。
* **私人配置路径：** `~/.bashrc` （当前登录用户的个人喜好。比如你喜欢敲 `ll` 代替 `ls -l`，这个指令就写在这里）。
* **核心操作：** 准备终端（Terminal）的交互环境，让你可以开始敲击键盘。
