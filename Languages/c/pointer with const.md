先直接回答你的问题：

- **它限制了什么？** 限制了你**不能通过指针 `a` 去修改它所指向的那个值**（也就是 `*a = 10;` 会报错）。
    
- **可以继续执行 `a = b` 吗？** **完全可以**。你可以随意改变指针 `a` 的指向，让它指向别的地址。

### 🧠 费曼记忆法：房卡与房间的隐喻

不要去死记硬背 `const` 和 `int` 的排列组合，我们在脑海里建立一个视觉场景：

- **指针 `a`** = 你手里拿着的**房卡**
    
- **指针指向的值 `*a`** = 房间里的**家具**
    

现在，我们把 `const` 想象成一张**“不可更改”的封条**。封条贴在哪里，哪里就不能动。

**1. `const int * a` （封条贴在房间上）**

- **场景：** 你有一张普通的房卡，但是房间门上贴了封条。
    
- **结果：** 你可以把手里的房卡扔了，换一张别的房间的卡（**`a = b` ✅ 允许**）。但是，不管你拿这宣卡开哪个房间，只要是用这张卡开的门，你都不能动房间里的家具（**`*a = 10` ❌ 报错**）。
    

**2. 强烈对比：`int * const a` （封条贴在房卡上）**

- **场景：** 房卡被强力胶水死死粘在了你的手上（房卡本身变成了 const）。
    
- **结果：** 你绝对没法换别的房卡了（**`a = b` ❌ 报错**）。但是，房间是没有封条的，你用这张焊死在手上的卡开门进去后，可以随便砸房间里的家具（**`*a = 10` ✅ 允许**）。

### 🔑 终极黑客口诀：看星号（`*`）的位置

如果你觉得房卡的例子还需要脑补，行业里有一句只看代码就能秒懂的四字真言：**“左物右指”**。

你就盯着代码里的星号 `*`，看 `const` 在它的哪边：

- **`const` 在 `*` 的左边** 👉 `const int * a`
    
    - **左物：** 限制的是**目标物体**。指针随意变，物体不准变。
        
- **`const` 在 `*` 的右边** 👉 `int * const a`
    
    - **右指：** 限制的是**指针本身**。物体随意变，指针不准变。
        

以后无论面试还是排查报错，只要看到代码，眼睛直接去找 `*`，瞬间就能想起来。
