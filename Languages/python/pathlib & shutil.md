### 模块一：`pathlib` (面向对象的路径处理核心)

_专注点：路径的拼接、解析、属性获取、单文件级别的增删改查。_

| **常用命令 / 属性**                                               | **💡 记忆方法 / 英文溯源**                                            | **功能描述**   | **详细参数说明**                                                         | **易混淆 / 反直觉点 (避坑指南)**                                                                |
| ----------------------------------------------------------- | ------------------------------------------------------------- | ---------- | ------------------------------------------------------------------ | ------------------------------------------------------------------------------------ |
| `Path(path)`                                                | **直译**：路径。万物皆由此起。                                             | 实例化路径对象    | `path`: 字符串格式的路径（绝对/相对）。                                           | **反直觉**：`Path` 对象是不可变的，修改路径操作会返回**新**对象。                                             |
| `/` (除号操作)                                                  | **形象化**：原生路径就是用 `/` 分隔的，这里直接把斜杠当“胶水”用。                        | 拼接路径       | `Path对象 / '字符串'`。                                                  | **极易踩坑**：若 `/` 右边是绝对路径（如 `C:\`），它会**覆盖**左边，直接返回右边。                                   |
| `.exists()`                                                 | **直译**：Exists（存在）。                                            | 检查路径是否存在   | 无参数。返回布尔值。                                                         | **注意**：如果路径存在但是一个损坏的软链接，会返回 `False`。                                                 |
| `.is_file()`<br><br>  <br><br>`.is_dir()`                   | **缩写还原**：Is File? / Is **Dir**ectory?                         | 判断是否为文件/目录 | 无参数。返回布尔值。                                                         | **反直觉**：即使后缀带 `.txt`，只要没在硬盘上实际创建，`.is_file()` 也返回 `False`。                           |
| `.mkdir()`                                                  | **缩写还原**：**m**a**k**e **dir**ectory（与Linux命令同名）。              | 创建目录       | `parents=False`: 建父目录；<br><br>  <br><br>`exist_ok=False`: 存在时是否报错。 | **黄金组合**：`.mkdir(parents=True, exist_ok=True)` 可避免99%的报错。                            |
| `.rglob()`<br><br>  <br><br>`.glob()`                       | **缩写还原**：**Glob**al (全局匹配) / **R**ecursive **Glob** (递归全局匹配)。 | 模式匹配查找     | `pattern`: 匹配模式，如 `'*.txt'`。                                       | **反直觉**：返回的是 **生成器** 而非列表，需套一层 `list()` 才能多次打印。                                      |
| `.unlink()`                                                 | **底层逻辑**：**un** (取消) + **link** (链接)。删文件在系统底层叫“断开硬盘链接”。       | 删除**单个文件** | `missing_ok=False`: 文件不存在时是否忽略报错。                                  | **限制**：只能删文件。删文件夹会报 `PermissionError`，需用 `shutil.rmtree()`。                          |
| `.parent`<br><br>  <br><br>`.parents`                       | **直译**：父母/长辈。向上找一层就是找爹，找所有祖先加 `s`。                            | 获取父目录      | `.parent` 是直接父级；`.parents[i]` 是第 i 级父级。                            | `.parents[0]` 等同于 `.parent`。越往上层索引越大。                                                |
| `.name`<br><br>  <br><br>`.stem`<br><br>  <br><br>`.suffix` | **形象化**：`name`是整根树枝，`stem`(词干)是主干(去尾)，`suffix`是尾巴(后缀)。        | 获取文件名属性    | **无需参数。**                                                          | `file.tar.gz` 中：`.name`=`file.tar.gz`, `.stem`=`file.tar` (只去最后后缀), `.suffix`=`.gz`。 |
| `.rename()`<br><br>  <br><br>`.replace()`                   | **直译**：Rename (重命名) / Replace (替换)。                           | 重命名/移动文件   | `target`: 目标路径。                                                    | **跨平台大坑**：Win下 `rename` 遇目标存在会报错；`replace` 则**强制覆盖**。建议统一定用 `.replace()`。            |
| `.read_text()`<br><br>  <br><br>`.write_text()`             | **直译**：读文本 / 写文本。带 _text 强调跳过繁琐的 `open()`。                    | 快捷读写文本     | `encoding=None`: 编码；<br><br>  <br><br>`data` (write专属)。            | `.write_text()` 会**直接覆盖**原文件，没有追加(`append`)模式。                                       |

### 模块二：`shutil` (高级文件操作工具)

_模块名溯源：**sh**ell **util**ities (Shell 实用工具)，意思是把操作系统终端（Shell）里的高级文件命令搬到了 Python 里。_

|**常用命令**|**💡 记忆方法 / 英文溯源**|**功能描述**|**详细参数说明**|**易混淆 / 反直觉点 (避坑指南)**|
|---|---|---|---|---|
|`copy()`|**直译**：复制。基础款。|复制文件（基础）|`src`: 源；`dst`: 目标。|**核心混淆**：它**不会**保留文件的创建/修改时间戳！|
|`copy2()`|**联想**：`2` 代表升级版，不仅复制内容，连“时间戳等元数据”也 `Too` (同音2) 复制过去。|完美复制文件|参数同 `copy()`。|**推荐**：日常无脑用 `copy2` 代替 `copy`，保留时间对排查很有帮助。|
|`rmtree()`|**缩写还原**：**r**e**m**ove **tree**。把目录当成一棵树，连根拔起。|递归删除目录|`path`: 目标；<br><br>  <br><br>`ignore_errors`: 忽略错误。|**极度危险**：等同Linux的 `rm -rf`。不会二次确认，不进回收站。<br><br>  <br><br>**权限坑**：遇只读文件会报错。|
|`move()`|**直译**：移动。|移动文件/文件夹|`src`: 源；`dst`: 目标。|**反直觉**：同磁盘是极速的重命名；**跨磁盘**移动则是先复制再删除原文件，很慢。|
|`copytree()`|**组合词**：**copy** + **tree**。复制整棵目录树。|递归复制目录|`src`: 源；`dst`: 目标；<br><br>  <br><br>`dirs_exist_ok=False`。|**经典报错**：目标目录若存在会崩溃，Python 3.8+ 须显式加 `dirs_exist_ok=True` 才能合并覆盖。|
|`make_archive()`|**组合词**：**make** (制作) + **archive** (归档/压缩包)。|打包/压缩目录|`base_name`: 包名(无后缀)；<br><br>  <br><br>`format`: 格式('zip')；<br><br>  <br><br>`root_dir`: 目录。|**易错点**：`base_name` (放哪) 和 `root_dir` (包谁) 的路径容易搞反，建议用绝对路径。|
|`unpack_archive()`|**组合词**：**unpack** (解包) + **archive** (归档)。|解压缩文件|`filename`: 包路径；<br><br>  <br><br>`extract_dir`: 解压目标。|**安全风险**：解压不可信的 `tar` 包，恶意路径可能导致“目录穿越”漏洞。|
|`disk_usage()`|**组合词**：**disk** (磁盘) + **usage** (使用量)。|查看磁盘空间|`path`: 目标挂载点/路径。|返回元组 `(total, used, free)`，单位是**字节 (Bytes)**，需自行除以 `1024**3` 转为 GB。|
