# Linux 开机自动化：全阶段实战 Demo 与逻辑拆解

本笔记按 Linux 启动的时间线先后顺序，提供了各阶段“开机自动化”的具体代码模板（Demo）。可直接拷贝、按需修改。

## 阶段一：底层存储挂载 (最高优先级)

**核心逻辑**：系统刚启动时，内核必须先拿到==“硬盘”和“内存盘”==，后续的程序才有地方读写文件。

**优先级**：**Highest (极高)**。在网络服务、Udev 和任何用户态程序启动之前执行。

**配置文件路径**：`/etc/fstab`

### 💻 Demo: 开机自动挂载 `tmpfs` 和指定分区

如果你想在内存中划出一块 50MB 的极速空间供业务程序读写（断电即焚），或者挂载额外的 eMMC/SD 卡数据分区，请将以下内容追加到 `/etc/fstab` 末尾：

```shell
# <file system>     <mount point>       <type>  <options>                   <dump>  <pass>
# 1. 挂载一个50MB的 tmpfs (内存盘)，主要用于存放高频读写的日志或核心库
tmpfs               /run/app_cache      tmpfs   defaults,size=50M,mode=1777 0       0

# 2. 挂载 SD 卡的第三个分区作为业务数据盘 (跳过开机坏块检查 pass=0)
/dev/mmcblk0p3      /mnt/userdata       ext4    defaults,noatime            0       0
```

> **修改指南**：
> 
> - `size=50M`：按需修改，务必确保不要超过板子的物理内存上限。
>     
> - 保存后，可输入 `mount -a` 立即测试挂载是否报错。
>     

## 阶段二：硬件热插拔与事件触发 (异步执行)

**核心逻辑**：基于底层的设备插拔事件（Uevent）来触发脚本。常用于工控机外接设备的自动初始化（如自动挂载 U 盘，或 4G 模块拨号）。

**优先级**：**High (高)**。只要内核检测到硬件总线状态变化，Udev 守护进程就会立即介入。

**配置文件路径**：`/etc/udev/rules.d/99-custom-devices.rules` （文件名需以 `.rules` 结尾）

### 💻 Demo: 插入 U 盘时自动运行挂载脚本

假设你希望现场人员插入 USB 麦克风或 U 盘时，系统能自动跑一段脚本：

**步骤 1：新建规则文件 `/etc/udev/rules.d/99-usb-automount.rules`**

Plaintext

```
# 当内核(KERNEL)发现 sd[a-z][0-9] (即USB存储分区) 添加(add)时，运行指定的脚本
ACTION=="add", KERNEL=="sd[a-z][1-9]", RUN+="/opt/scripts/usb_on_insert.sh %k"
```

**步骤 2：编写被触发的执行脚本 `/opt/scripts/usb_on_insert.sh`**

Bash

```
#!/bin/sh
# $1 会接收到 Udev 传过来的 %k 参数（如 sda1）
DEVICE=$1
MOUNT_POINT="/mnt/usb_${DEVICE}"

# 创建挂载点并挂载
mkdir -p ${MOUNT_POINT}
mount /dev/${DEVICE} ${MOUNT_POINT}

# 记录日志 (由于 Udev 脚本环境极为精简，务必使用绝对路径输出日志)
echo "$(date): USB Device ${DEVICE} auto-mounted to ${MOUNT_POINT}" >> /var/log/usb_hotplug.log
```

> **修改指南**：确保脚本具有可执行权限 `chmod +x /opt/scripts/usb_on_insert.sh`。修改完 rules 文件后，执行 `udevadm control --reload-rules` 生效。

## 阶段三：核心业务守护进程 (依赖式执行)

**核心逻辑**：这是拉起业务应用（如数据采集程序、Web 服务）的最标准方式。由 Systemd 管理，可以精确控制“谁先启、谁后启”，死了还能自动重启。

**优先级**：**Medium (中)**。由依赖图（Dependency Graph）决定，通常等网络或特定硬件就绪后执行。

**配置文件路径**：`/etc/systemd/system/my_business.service`

### 💻 Demo: 注册一个崩溃自动重启的业务进程

假设你写了一个音频采集的 C 程序 `/usr/bin/audio_capture`，需要开机自动跑在后台：

**新建文件 `/etc/systemd/system/audio_capture.service`**

Ini, TOML

```
[Unit]
Description=Industrial Audio Capture Service
# 依赖声明：必须等网络服务起来，且系统的本地挂载点都挂载完毕后才执行
After=network.target local-fs.target

[Service]
# 设置程序的运行工作目录
WorkingDirectory=/opt/audio_app
# 核心启动命令 (必须是绝对路径，如果是脚本要在前面加 /bin/bash)
ExecStart=/usr/bin/audio_capture --mode industrial

# 守护逻辑：如果程序崩溃(退出码非0)，5秒后自动重启
Restart=on-failure
RestartSec=5

# 环境变量：让程序优先去 tmpfs 找动态库
Environment="LD_LIBRARY_PATH=/run/app_cache:$LD_LIBRARY_PATH"
# 限制日志输出级别
Environment="LOG_LEVEL=INFO"

[Install]
# 挂载到多用户运行级别(标准的命令行启动完成阶段)
WantedBy=multi-user.target
```

> **修改指南**：
> 
> 1. 保存后，重新加载配置：`systemctl daemon-reload`
>     
> 2. 设置开机自启：`systemctl enable audio_capture.service`
>     
> 3. 手动启动测试：`systemctl start audio_capture.service`
>     

## 阶段四：开机兜底执行 (晚期简单脚本)

**核心逻辑**：当系统中所有的基础服务（网络、SSH、系统日志）都已经完全启动完毕后，内核会最后看一眼这个文件，执行里面的简单顺手命令。

**优先级**：**Low (低)**。属于 Boot 阶段的最末尾。

**配置文件路径**：`/etc/rc.local` (如果系统默认没有，可直接新建该文件)

### 💻 Demo: 杂项初始化与 GPIO 导出

如果有些操作太简单，不值得专门写一个 Systemd service（比如点亮一个状态指示灯，或者给某个驱动文件喂个参数），就写在这里。

**编辑 `/etc/rc.local`**

Bash

```
#!/bin/sh -e
# -e 表示遇到错误立刻停止运行

# 1. 硬件外设初始化：导出 i.MX6ULL 的某个 GPIO 引脚并设置为高电平
# 假设导出 GPIO4_14 (编号根据具体芯片手册计算)
echo 110 > /sys/class/gpio/export
echo out > /sys/class/gpio/gpio110/direction
echo 1 > /sys/class/gpio/gpio110/value

# 2. 内存清理机制调整：降低系统 swap 倾向，保证业务实时性
echo 10 > /proc/sys/vm/swappiness

# 3. 顺手拉起一个不需要守护的轻量级后台脚本 (必须加 & 放后台，否则会卡死整个开机流程)
/opt/scripts/simple_watchdog.sh &

# 脚本必须以 0 正常退出
exit 0
```

> **修改指南**：务必检查 `/etc/rc.local` 是否有执行权限：`chmod +x /etc/rc.local`。**切记：里面任何阻塞性质的命令（死循环等），都必须在行尾加 `&` 放入后台。**

## 阶段五：用户会话交互 (登录触发)

**核心逻辑**：只有当开发人员通过串口（UART）或 SSH 输入账密成功登录后，才会执行。如果不登录，这里的代码永远不会被触发。

**优先级**：**None (非系统级)**。属于用户空间。

**配置文件路径**：`~/.bashrc` (针对当前用户) 或 `/etc/profile` (针对所有用户)

### 💻 Demo: 开发者环境与快捷命令配置

**追加到 `~/.bashrc` 末尾**

Bash

```
# 1. 注入交叉编译或特定工具链的路径
export PATH="/opt/gcc-arm-linux-gnueabihf/bin:$PATH"

# 2. 定义工程调试用的快捷命令 (Alias)
alias ll='ls -alF'
alias logs='journalctl -u audio_capture.service -f'
alias reset_audio='systemctl restart audio_capture.service'

# 3. 登录时打印一句系统状态提示 (比如查当前板子的温度)
if [ -f /sys/class/thermal/thermal_zone0/temp ]; then
    cpu_temp=$(cat /sys/class/thermal/thermal_zone0/temp)
    # 将千分之一度转换为度
    cpu_temp_c=$(awk "BEGIN {print $cpu_temp/1000}")
    echo -e "\e[32m[System Ready] Current CPU Temp: ${cpu_temp_c}°C\e[0m"
fi
```
