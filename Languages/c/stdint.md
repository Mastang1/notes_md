# WHY:
#### ==因为不同bits的cpu的compiler基本总是按照cpu位宽定义int的宽度==
##### 不全对:1. C语言总公司（ISO标准）有个死规定：`int` 最小也得能存下 -32767 到 32767（也就是至少得是 16 位）
##### 2. 为了向下兼容，现在的 64 位 Windows 和 Linux 系统，都默契地把 int 强行保留在了 32 位，而只把指针和 long 类型扩展到了 64 位。

### 导致的问题：代码中int长度就不能确定了，跨平台可能会有问题

### 解决方法： 写一个stdint.h作为适配层，不同bits的CPU不同，保证uint32_t之类的表达式在不同平台都是指定的宽度
