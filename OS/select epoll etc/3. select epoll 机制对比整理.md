# 一、select vs epoll 性能对比（工程级角度）

## 1️⃣ 时间复杂度

### select

每次唤醒后：

```text
for 所有 fd:
    file->poll()
```

复杂度：

```text
O(n)
```

n = 监听的 fd 总数

无论有 1 个 ready 还是 100 个 ready，都要扫全部。

---

### epoll

事件发生时：

```text
只把 ready 的 fd 放入 ready list
```

epoll_wait 时：

```text
只遍历 ready list
```

复杂度：

```text
O(ready_count)
≈ O(1)（当 ready 很少时）
```

---

# 2️⃣ 注册成本对比

### select

每次调用都要：

- 遍历 fd
    
- 调 poll
    
- 注册 wait_queue
    

每轮都重新做一遍。

---

### epoll

调用一次：

```c
epoll_ctl(ADD)
```

之后：

- 持久注册
    
- 不再重复扫描全部 fd
    

注册成本从“每次调用”变成“一次性”。

---

# 3️⃣ 内存拷贝成本

### select

每次调用：

- 用户态 fd_set → 拷贝到内核
    
- 内核修改 → 再拷贝回用户态
    

有两次内存复制。

---

### epoll

epoll_wait 只返回 ready 数组。

减少大规模 bitset 拷贝。

---

# 4️⃣ 当 fd 很少时谁快？

重要现实结论：

|场景|更合适|
|---|---|
|fd < 50|select 差异不明显|
|fd 数千|epoll 明显优势|
|高并发服务器|必须 epoll|

原因：

epoll 有额外结构（红黑树 + ready list），小规模下优势不明显。

---

# 5️⃣ 真正的性能差异来源

不是调度。

不是 wake_up。

而是：

```text
select = 唤醒后扫描全部
epoll  = 事件发生时登记
```

把成本从：

```text
“唤醒时 O(n)”
```

移动到：

```text
“事件发生时 O(1) 插入 ready list”
```

这才是架构级差异。

# 二、压缩版结构笔记（select vs epoll）

下面是可以长期记忆的版本。

---

# 一句话核心区别

```text
select：唤醒后扫描全部 FD
epoll ：内核维护 ready list，只返回 ready FD
```

---

# 1️⃣ select 机制（核心）

流程：

1. 每次调用重新遍历所有 fd
    
2. file->poll 注册 wait_queue
    
3. 若无 ready → schedule()
    
4. 被 wake_up
    
5. 再次遍历所有 fd
    
6. 返回 ready 集合
    

特点：

- 无事件缓存
    
- 每次重新注册
    
- O(n)
    

---

# 2️⃣ epoll 机制（核心）

分为两阶段：

### 阶段 A：注册阶段

```c
epoll_ctl(ADD)
```

- fd 加入 interest list
    
- 注册回调
    
- 持久化
    

---

### 阶段 B：事件发生阶段

当 fd ready：

- 内核回调
    
- fd 加入 ready list
    
- 唤醒 epoll_wait
    

epoll_wait：

- 只遍历 ready list
    
- 返回 ready fd
    

特点：

- 有 ready list（事件缓存）
    
- 持久注册
    
- O(ready_count)
    

---

# 3️⃣ 关键结构差异

|结构|select|epoll|
|---|---|---|
|interest list|无|有|
|ready list|无|有|
|每轮注册|是|否|
|唤醒后扫描|全部 fd|仅 ready fd|

---

# 4️⃣ 性能本质总结

select 慢在：

```text
唤醒后 O(n) poll()
```

epoll 快在：

```text
事件发生时 O(1) 插入 ready list
```

不是因为：

- 睡眠更高级
    
- 调度更快
    

而是因为：

```text
数据结构不同
```

---

# 5️⃣ 用 RTOS 思维记忆

select：

```text
任务醒来后遍历所有资源确认
```

epoll：

```text
资源发生事件时主动登记
任务只消费登记结果
```

---

# 最终工程级理解

select：

```text
没有事件缓存结构
```

epoll：

```text
多了 ready list（事件缓存）
```

这就是本质差异。
